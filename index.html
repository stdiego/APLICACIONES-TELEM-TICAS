Especificación Técnica y Normativa

El desarrollo se rige por estándares de ingeniería para garantizar interoperabilidad telemática y rendimiento. Se define el hardware de despliegue y los protocolos de comunicación utilizados.

Requisitos de Infraestructura (Host)

Fundamentación y Protocolos

HTTP/1.1 (RFC 2616):

Protocolo de capa de aplicación sin estado. Se utiliza arquitectura RESTful teórica para las peticiones GET/POST del formulario de login.

ACID (Base de Datos):

Garantía de Atomicidad, Consistencia, Aislamiento y Durabilidad en transacciones MySQL, crítico para la integridad de datos de usuarios.

IEEE 830 (SRS):

Adopción de buenas prácticas de especificación de requisitos de software (Funcionales y No Funcionales).

Arquitectura del Sistema (LAMP)

Implementación de una Arquitectura de 3 Capas (Three-Tier), desacoplando la lógica de presentación (Client), la lógica de negocio (App Server) y el acceso a datos (Persistence).

Capa Cliente

Frontend: HTML5 Semántico + CSS3. Responsable de la UI y validación de formularios pre-envío.

Capa Lógica

Backend: PHP 7.4 sobre Apache Web Server. Gestiona sesiones, cookies y sanitización de inputs.

Capa de Datos

Persistencia: MySQL (InnoDB Engine). Almacena credenciales y perfiles en esquema relacional.
